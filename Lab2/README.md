# wmpf  

Туренко Юлія

Лабораторна робота №2

Мета: Ознайомитися та попрактикуватися з розробкою серверного застосунку на Node.js з використанням Express та React.


Індивідуальне завдання №1: Створіть базову соціальну мережу на Node.js з використанням Express. Реалізуйте фронтенд на React для відображення користувачів та їх постів.

Хід вирішення: 

Індивідуальне завдання №2: Додайте можливість створювати та коментувати пости. Реалізуйте систему дружніх зв'язків між користувачами.

Хід вирішення: Для користувачів була створена сутність User, яка містить список друзів. Було реалізовано створення користувачів, додавання і відображення друзів. Для реалізації постів було створено окрему сутність Post, яка містить ідентифікатор автора, текст поста й дату створення. Було реалізовано API для отримання списку постів і створення нового поста. Для коментарів була створена сутність Comment, які прив’язуються до конкретного поста через postId. У результаті користувачі можуть створювати пости та додавати коментарі до них, а також будувати дружні зв’язки один з одним.

Індивідуальне завдання №3: Реалізуйте систему особистих повідомлень між користувачами. Додайте систему пошуку користувачів та постів.

Хід вирішення: Для реалізації повідомлень були створені сутності: Chat (представляє чат між користувачами), ChatUser (реалізує зв’язок many-to-many між чатами та користувачами) і Message.Було реалізовано автоматичне створення приватного чату, відкриття чатів, надсилання повідомлень та їх отримання. Frontend реалізує окрему сторінку месенджера зі списком друзів та областю повідомлень. Для реалізації пошуку користувачів та постів було використано фільтрацію масивів на frontend.

Індивідуальне завдання №4: Використовуйте компресію (gzip) для передачі даних з сервера на клієнтський інтерфейс для зменшення об'єму даних, які передаються по мережі.

Хід вирішення: Для зменшення об’єму даних, які передаються між сервером та клієнтом, було використано middleware compression для Express. Middleware автоматично стискає HTTP-відповіді за допомогою алгоритму gzip, що покращує швидкість завантаження та зменшує мережевий трафік. Варто зазначити, що Express виконує компресію не для всіх відповідей. Невеликі за об’ємом JSON-відповіді можуть не стискатися автоматично, оскільки це не дає суттєвого виграшу у розмірі даних.
