Практичне заняття №2

Мета: Робота із форматом даних json, робота із запитами - метод fetch(). React.

Індивідуальне завдання №1: Список покупок:
•  Створіть файл HTML з елементом <script>.
•  Використовуйте fetch для завантаження JSON-файлу зі списком покупок.
•  Розберіть JSON-файл за допомогою JSON.parse().
•  Створіть список HTML з елементами <li> для кожного пункту списку покупок.
 
Хід вирішення: було створено базову структуру HTML-документа з контейнером для списку. За допомогою методу fetch реалізовано запит до локального файлу data.json. Отримані дані у вигляді тексту перетворено в JavaScript-об'єкт за допомогою JSON.parse(). Далі, шляхом ітерації (перебору) масиву, для кожного елемента динамічно створено тег <li>, який додано до батьківського списку на сторінці за допомогою appendChild.

Індивідуальне завдання №2: Напишіть програму, яка приймає JSON-файл та значення фільтра в якості вхідних даних. Програма повинна фільтрувати дані JSON, залишаючи лише ті елементи, які відповідають заданому значенню фільтра. Виведіть відфільтровані дані JSON.


Хід вирішення: було використано метод filter(), який перевіряє кожну одиницю даних на відповідність значенню фільтра. Результат фільтрації зберігається у новому масиві, який згодом виводиться на екран.

Індивідуальне завдання №3: Розробіть веб-додаток для обміну повідомленнями між користувачами. Додайте можливість створення чат-кімнат та відправлення повідомлень.

Хід вирішення: спочатку було створено необіхдні компоненти: App (зберігає та завантажує всі дані й передає їх у компоненти), MessageList (показує повідомлення тільки вибраної кімнати і дозволяє додавати нові) і RoomList (відображає список кімнат і дає можливість їх створення). Для відкриття певної кімнати у App є currentRoom, тобто id вибраної кімнати, який змінюється через setCurrentRoom і відповідний onClick у MessageList. У App додано перевірку на наявність кімнат і чи обрана певна, щоб відображати відповідне повідомлення замість повідомлень (MessageList). Загалом для відображення списків використовується map(), для збереження/завантаження - useEffect, для керування станом та оновлення інтерфейсу - useState.
Для збереження даних використано localStorage, що дозволяє працювати з ними у форматі JSON без використання бази даних. Одразу ж при старті програми зчитуються усі дані з localStorage, а при кожній зміні - серіалізуються.

Індивідуальне завдання №4: Додайте можливість фільтрації, сортування і пошуку.

Хід вирішення: функціонал пошуку, сортування і фільтрації був реалізований для кімнат (у RoomList) за допомогою вбудованих функцій filter, includes і sort. Пошук по назві, сортування за датою створення, фільтрація за типом (public/private).


Результат: реалізовано просту сторінку HTML, яка завантажує дані з JSON-файлу, перетворює їх у JavaScript-об’єкти та відображає на сторінці у вигляді структурованого списку з можливістю фільтрації; розроблено веб-додаток на React з використанням TypeScript, який реалізує обмін повідомленнями між користувачами з можливістю створення чат-кімнат і функціоналом пошуку, фільтрації та сортування.
