# wmpf  

Туренко Юлія
ПЗПІ-23-4

Лабораторна робота №4

Мета: Створення складної бази даних.


Індивідуальне завдання №1: Розробіть базу даних для соціальної мережі з таблицями: Користувачі, Друзі, Пости, Коментарі, Лайки. Визначте зв'язки та обмеження для забезпечення цілісності даних.

Хід вирішення: Було спроєктовано реляційну структуру бази даних, що складається з основних сутностей соцмережі та зв'язків. За допомогою Entity Framework Core налаштовано первинні та зовнішні ключі, а також каскадне видалення залежних об'єктів. Для оптимізації майбутніх пошукових запитів та забезпечення унікальності даних було створено відповідні індекси та обмеження (Constraints).

Індивідуальне завдання №2: Створіть моделі бази даних соціальної мережі. Реалізуйте додавання нових постів та коментарів.

Хід вирішення: На основі спроєктованої схеми створено відповідні Entity-класи (User, Post, Comment тощо) з відповідними їм властивостями. Реалізовано класи Repository для ізоляції логіки доступу до даних, що дозволило безпечно додавати нові пости та коментарі через асинхронні методи. Створено відповідні контролери, які приймають дані від клієнта у форматі JSON, валідують їх та зберігають зміни в базі даних.

Індивідуальне завдання №3: Впровадьте функціонал пошуку та фільтрації постів у соціальній мережі.

Хід вирішення: В репозиторії постів було розроблено LINQ-запит для отримання всіх постів, окремо створено запит GetPostsBySearchQuery, що отримує пошукову строку і на основі слів здійснює пошук за Content постів. Так само реалізовано пошук користувачів за їх UserName. Розроблені запити було інтегровано в ендпоінти.

Індивідуальне завдання №4: Розробіть систему логування та моніторингу запитів у соціальній мережі. Зберігайте логи у базі даних та надайте інтерфейс для їх перегляду адміністраторами.

Хід вирішення: Створено модель RequestLog для збереження технічних метрик запитів, включаючи HTTP-метод, шлях, статус-код, ID користувача та час виконання в мілісекундах. Було розроблено кастомне проміжне ПЗ (Middleware), яке в автоматичному режимі перехоплює кожен вхідний запит, вимірює його швидкість та логує дані в БД через спеціальний репозиторій. Для адміністраторів створено окремий API-контролер з підтримкою пагінації, що надає інтерфейс для моніторингу загальних, повільних або специфічних для конкретного користувача логів.
