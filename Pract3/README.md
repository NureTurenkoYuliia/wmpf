Практичне заняття №3

Мета: Мобільна розробка на Android з Kotlin.

Індивідуальне завдання №1: Реалізуйте кнопку, при натисканні на яку буде виводитися повідомлення "Вітаємо з першим додатком на Kotlin!".
 
Хід вирішення: Було створено простий мобільний екран із кнопкою. Після натискання кнопки за допомогою механізму обробки подій setOnClickListener відображається повідомлення через компонент Toast. Для побудови інтерфейсу використано XML-файл activity_main, а логіка роботи реалізована у MainActivity.

Індивідуальне завдання №2: Розробіть додаток для ведення списку завдань (ToDo). Користувач повинен мати можливість додавати та відмічати завдання виконами.

Хід вирішення: Було реалізовано додаток для ведення списку завдань (ToDo List). Користувач має можливість вводити текст завдання, додавати його до списку та позначати виконані завдання. Для реалізації списку використано компонент RecyclerView, який дозволяє ефективно відображати велику кількість елементів. Для зв’язку даних зі списком було створено власний клас TaskAdapter, який наслідується від RecyclerView.Adapter. Дані про завдання зберігались у моделі Task, створеній за допомогою data class.

Індивідуальне завдання №3: Реалізуйте функціонал відправлення запиту до віддаленого сервера на отримання та відображення даних. Використайте Retrofit та Kotlin Coroutines.

Хід вирішення: Було реалізовано взаємодію із віддаленим сервером через HTTP-запити. Для цього використано бібліотеку Retrofit, яка спрощує роботу з REST API. Дані отримувались із тестового сервера у форматі JSON та автоматично перетворювались у Kotlin-об’єкти за допомогою Gson Converter. Для асинхронного виконання запитів використано Kotlin Coroutines та lifecycleScope.launch, що дозволило виконувати мережеві операції без блокування головного потоку інтерфейсу.

Індивідуальне завдання №4: Створіть додаток для зберігання та обміну рецептами. Кожен рецепт повинен включати назву, інгредієнти та інструкції приготування.

Хід вирішення: було створено додаток для збереження та перегляду рецептів. Реалізовано можливість: додавання рецептів, перегляду списку рецептів, відкриття детальної інформації про рецепт. Для локального збереження даних використано Room Database — ORM-бібліотеку для роботи з SQLite у Android. Було створено: сутність Recipe, DAO-інтерфейс RecipeDao, клас бази даних RecipeDatabase.

Для навігації між екранами використано Intent. Дані між Activity передавались через putExtra() та getStringExtra(). Також у проєкті було реалізовано головне меню (MenuActivity), яке забезпечує навігацію між усіма виконаними завданнями. Це дозволило запускати кожну частину роботи окремо в межах одного Android-додатка.


Результат: було розроблено Android-додаток мовою Kotlin із використанням XML-інтерфейсів та Android SDK у середовищі Android Studio.
